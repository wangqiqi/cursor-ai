# Reader Test · L2 语义验收

L1 脚本绿 **不** 等于读者能跟做。发版档建议做 Reader Test。

## 何时做

- 新手册首版
- 大改版（故事线重组、图 >30% 更换）
- contract 含 `inferred: true` 故事线
- plan `DOC-*` 阶段 3

用户说「不用测读者」→ 记录跳过；仍建议维护者通读一遍。

## 方法

1. 准备**仅**手册 Markdown（+ 图路径说明），不给实现代码
2. 委派 **review** agent（只读）或新会话
3. 逐题提问（下方模板）
4. 答不出 / 歧义 / 与图不符 → 回改对应节 · 必要时 regen 单张图

## 十问模板（可按产品删减）

| # | 问题 | 通过标准 |
|---|------|----------|
| 1 | 第一次培训应从哪条故事线开始？ | 与 §读者引导一致 |
| 2 | 开始前必须检查哪几项？ | 与前提清单一致 |
| 3 | 角色 A 登录后默认看到什么？ | 与角色表一致 |
| 4 | 故事线 X 第 3 步点哪里？ | 与步骤表 + 图号一致 |
| 5 | 空数据时走哪条线？ | 有空态故事线 |
| 6 | 图 N 红框标的是什么？ | 与步骤叙述一致 |
| 7 | 做完故事线 Y 应看到什么？ | 与期望结果一致 |
| 8 | 手册是否依赖外链才能操作？ | 可单独发布则否 |
| 9 | 演示/测试账号从哪里读？ | 指向 env/管理员分配，非 skill 正文 |
| 10 | 本版相对上版最大变化？ | 与版本头/CHANGELOG 一致 |

## 输出格式

```text
题号 · 通过|失败 · 歧义点 · 建议改哪一节
```

全通过 → 可标 manual / DOC ✅。有失败 → 不得标 ✅。
